# install vite

- **Commit:** `1d54a2a`
- **Datum:** 2026-04-29
- **Autor:** Oliver Jung

## Worum geht es?

Dies ist der erste inhaltliche Commit des Projekts. Hier wird ein **Vite**-Projekt mit **TypeScript** aufgesetzt. Vite ist ein moderner Build- und Entwicklungsserver, der besonders schnelle Reloads ermöglicht. Es entstehen die grundlegenden Dateien: eine `index.html` als Einstiegspunkt, eine `package.json` mit den Skripten und Abhängigkeiten sowie eine `tsconfig.json` für die TypeScript-Konfiguration.

## Die Änderungen im Detail

### `index.html` – der Einstiegspunkt

Vite verwendet die `index.html` als zentralen Startpunkt. Wichtig ist das `<div id="app">`, in das später die Anwendung gerendert wird, und das `<script type="module">`, das die TypeScript-Einstiegsdatei lädt.

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Hotelanwendung</title>
  </head>
  <body>
    <div id="app"></div>
    <script type="module" src="/src/main.ts"></script>
  </body>
</html>
```

### `package.json` – Skripte und Abhängigkeiten

Hier werden die drei klassischen Vite-Skripte definiert: `dev` startet den Entwicklungsserver, `build` erzeugt eine Produktionsversion (vorher prüft `tsc` die Typen) und `preview` zeigt den Build lokal an.

```json
{
  "name": "hotelanwendung",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "tsc && vite build",
    "preview": "vite preview"
  },
  "devDependencies": {
    "typescript": "~6.0.2",
    "vite": "^8.0.10"
  }
}
```

### `tsconfig.json` – TypeScript-Konfiguration

Die Konfiguration setzt auf den modernen „Bundler"-Modus, der gut mit Vite zusammenarbeitet. `noEmit` ist gesetzt, weil nicht `tsc`, sondern Vite den eigentlichen Build übernimmt; `tsc` dient nur der Typprüfung.

```json
{
  "compilerOptions": {
    "target": "es2023",
    "module": "esnext",
    "lib": ["ES2023", "DOM"],
    "types": ["vite/client"],
    "skipLibCheck": true,
    "moduleResolution": "bundler",
    "allowImportingTsExtensions": true,
    "verbatimModuleSyntax": true,
    "moduleDetection": "force",
    "noEmit": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "erasableSyntaxOnly": true,
    "noFallthroughCasesInSwitch": true
  },
  "include": ["src"]
}
```

Die Datei `src/main.ts` wird zunächst leer angelegt – sie wird im nächsten Commit mit Inhalt gefüllt.

## Was wurde erreicht?

Das Projekt ist initialisiert und lässt sich mit `npm run dev` starten. Die technische Grundlage (Vite + TypeScript) steht.
