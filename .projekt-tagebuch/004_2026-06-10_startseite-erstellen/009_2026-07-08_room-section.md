# room section

- **Commit:** `aa199c3`
- **Datum:** 2026-07-08
- **Autor:** Oliver Jung

## Worum geht es?

Die Startseite bekommt einen neuen Bereich: die **Zimmer-Vorstellung** („Zimmer mit Weitblick"). Er zeigt eine Beispiel-Suite mit Bild, Preis, Beschreibung und einer Liste der Ausstattungsmerkmale (King-size Bett, Bad, Klima/TV, Gym-Zugang) sowie einem „Buchen"-Button. Dazu wird das Farb-Theme um einen hellen Lila-Ton ergänzt.

## Die Änderungen im Detail

### `src/style.css` – neue Theme-Farbe

Für den Hintergrund des Zimmer-Bereichs wird eine helle Variante der Hausfarbe definiert. Sie steht danach als Utility-Klasse `bg-purple-haze-light` zur Verfügung.

```diff
 @theme {
     --color-purple-haze: #642360;
+    --color-purple-haze-light: #f1e8e8;
     --color-purple-haze-dark: #381436;
     --color-eggshell: #fffcf6;
```

### `src/views/HomeView/Home.ts` – Assets importieren

Für die Suite und ihre Ausstattungs-Icons werden neue Bild-Importe ergänzt. Über den Vite-Import erhält jede Datei eine gebündelte URL, die dann im Template als `src` genutzt wird.

```diff
 import fam_stroheim from '../../assets/img/familie-stroheim.jpg';
+import double_suite from '../../assets/img/double-suite.jpg';
+import king_bed from '../../assets/img/icons/kingsize.svg';
+import bathroom from '../../assets/img/icons/bathroom.svg';
+import klima_tv from '../../assets/img/icons/klima.svg';
+import gym from '../../assets/img/icons/gym.svg';
```

### `src/views/HomeView/Home.ts` – der neue Zimmer-Abschnitt

Am Ende des Templates wird eine `<section>` mit hellem Hintergrund (`bg-purple-haze-light`) angehängt. Sie besteht aus einer Überschrift, einer Karte mit dem Zimmerbild und dessen Beschreibung sowie einer Liste der Ausstattungsmerkmale. Auf kleinen Bildschirmen liegt alles untereinander (`flex-col`), ab dem `768`-Breakpoint nebeneinander (`768:flex-row`).

```html
<section class="bg-purple-haze-light flex flex-col gap-y-6 py-10 px-4">
    <div class="flex flex-col gap-y-3 456:gap-y-0 items-center">
        <h2 class="font-playfair-display text-24 456:text-32 768:text-48 text-purple-haze-dark font-medium text-center">Zimmer mit Weitblick</h2>
        <p class="w-46 456:w-full font-antic-didone text-16 768:text-20 text-purple-haze-dark text-center leading-5">Exklusive Rückzugsorte zwischen Bergen und See</p>
    </div>

    <div class="bg-white flex flex-col 768:flex-row gap-y-6">
        <div>
            <img src="${double_suite}" alt="Double Suite">
            <p class="px-4 font-antic-didone text-16 text-purple-haze-dark">231€ Nacht/ p. P.</p>
        </div>

        <div class="flex flex-col gap-y-4 px-4">
            <h3 class="font-playfair-display text-20 text-purple-haze-dark">Double Suite</h3>
            <p class="font-antic-didone text-16 text-purple-haze-dark">Edles Design, viel Raum und der Blick in die Karawanken machen sie zum idealen Rückzugsort für zwei.</p>
        </div>

        <div class="flex flex-col gap-y-4 pb-3">
            <div class="flex gap-y-2 px-4">
                <img src="${king_bed}" class="w-9.75 h-6" alt="King- size Bett">
                <span class="font-antic-didone text-16 text-purple-haze-dark">King- size Bett</span>
            </div>
            <!-- weitere Ausstattungs-Zeilen: bathroom, klima_tv, gym -->
        </div>
    </div>

    <button class="bg-purple-haze self-center py-2 px-3 rounded-md text-white">Buchen</button>
</section>
```

Jedes Ausstattungsmerkmal folgt demselben Muster: ein `<div class="flex">` mit einem Icon (`<img>`) und einem beschreibenden `<span>` daneben. Das macht den Code gut wartbar, weil sich alle Zeilen gleich aufbauen.

## Was wurde erreicht?

Die Startseite besitzt nun einen vollständigen **Zimmer-Bereich** als eigenständige Sektion. Die Ausstattung wird mit Icons visuell aufbereitet, und über den „Buchen"-Button entsteht ein erster Call-to-Action. Im nächsten Commit wird dieser Bereich typografisch und im Layout überarbeitet.
